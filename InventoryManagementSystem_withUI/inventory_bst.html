Insights

✅ Best Case — Balanced BST

Items inserted in random order. Tree stays balanced. Max levels = log₂(n). Example: 7 items → only 3 levels.


⚠️ Worst Case — Skewed BST

Items inserted in sorted order (e.g. Apple → Banana → Cherry). Tree becomes a linked list. Search = O(n)!


🔧 Advanced Fix (Theory)

AVL Tree auto-balances on every insert/delete.
Red-Black Tree: used in C++ map and Java TreeMap. Both guarantee O(log n) always.
